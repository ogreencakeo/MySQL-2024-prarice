[ 스토어드 프로시저 (Stored Procedure) ]
스토어드 프로그램은 데이터베이스에서 자주 사용되는 복잡한 작업을 간단한 호출로 실행할 수 있도록 
미리 정의된 SQL 코드의 모음입니다. 
이러한 프로그램은 스토어드 프로시저, 스토어드 함수, 트리거, 커서 등으로 구성됩니다. 
각각의 목적과 용도가 있지만, 여기서는 스토어드 프로시저에 대해 자세히 알아보겠습니다.

[ 개요 ]
스토어드 프로시저는 데이터베이스 내에 저장된 프로그램으로, SQL 명령문의 그룹을 하나의 단위로 묶어서 실행할 수 있습니다.
이를 사용하면 자주 사용되는 복잡한 쿼리를 단일하게 관리하고 호출할 수 있습니다.

[ 사용 예시 ]
스토어드 프로시저를 호출할 때는 CALL 프로시저_이름() 형식을 사용합니다.
이는 데이터베이스를 모듈화하여 필요할 때마다 호출할 수 있어서 
MySQL 등의 데이터베이스 시스템을 효율적으로 운용할 수 있게 도와줍니다.

[ 스토어드 프로시저의 형식 ]
1. 매개변수와 오류처리
- 스토어드 프로시저는 입력 매개변수와 출력 매개변수를 사용하여 작성됩니다.

- 입력 매개변수를 정의할 때는 IN 키워드를 사용하여 매개변수 이름과 데이터 형식을 지정합니다.
 IN 입력매개변수이름 데이터형식

-출력 매개변수를 정의할 때는 OUT 키워드를 사용하여 출력 매개변수 이름과 데이터 형식을 지정합니다.
 OUT 출력매개변수이름 데이터형식

[ 매개변수를 사용한 스토어드 프로시저 실행 ]
- 입력 매개변수가 있는 스토어드 프로시저를 실행할 때는 다음과 같이 호출합니다.
 CALL 프로시저이름(전달값)

- 출력 매개변수가 있는 스토어드 프로시저를 실행하고, 결과를 사용하기 위해서는 다음과 같이 호출합니다.
CALL 프로시저이름(@변수명);
 SELECT @변수명;
