[ 인덱스 생성 방법 ]
기본 문법: 
CREATE INDEX 인덱스명 ON 테이블명 (열);

예시: 
CREATE INDEX idx_name ON employees (last_name);

[ 보조 인덱스 생성과 고유 옵션 ]
1. 보조 인덱스(secondary index)는 테이블의 특정 열에 대한 인덱스를 생성합니다.
2. UNIQUE 옵션은 중복을 허용하지 않는 고유한 인덱스를 생성할 때 사용합니다.

[ 클러스터형 인덱스 ]
일부 데이터베이스 시스템에서는 클러스터형 인덱스를 지원하며, 테이블의 데이터 자체를 정렬된 형태로 저장합니다. 
CREATE INDEX로는 클러스터형 인덱스를 생성할 수 없습니다.

[ 인덱스 삭제 ] 
인덱스를 삭제할 때는 보조 인덱스부터 삭제해야 합니다.

문법:
DROP INDEX 인덱스명 ON 테이블명;

[ 인덱스 조회 ]
SHOW INDEX FROM 테이블명; 
명령을 사용하여 특정 테이블의 인덱스 정보를 조회할 수 있습니다.

[ 복합 인덱스 생성 ]
두 개 이상의 열을 조합하여 인덱스를 생성하는 것으로, 복합 인덱스는 여러 열에 대한 검색을 최적화할 수 있습니다.

[ 풀 테이블 스캔(Full Table Scan) ]
인덱스를 사용하지 않고 전체 테이블을 순차적으로 스캔하는 것을 의미합니다. 이는 성능 저하를 초래할 수 있으므로 피해야 합니다.

[ 인덱스 생성 판단시 고려사항 ]
1. 인덱스는 WHERE 절에서 자주 사용되는 열에 생성하는 것이 좋습니다.
2. 데이터 중복이 높은 열에는 인덱스를 만들어도 성능 향상 효과가 미비할 수 있습니다.
3. 외래키에 설정된 열에는 자동으로 외래키 인덱스가 생성됩니다.
4. 조인에 자주 사용되는 열에도 인덱스를 생성하는 것이 성능에 도움이 됩니다.
5. 사용하지 않는 인덱스는 성능 저하의 요인이므로 제거하는 것이 좋습니다.

위의 사항들은 데이터베이스 성능 최적화를 위해 인덱스를 어떻게 생성하고 관리해야 하는지에 대한 기본적인 지침을 제공합니다. 
실제 환경에서는 데이터베이스 시스템의 특성과 쿼리 패턴에 따라 세부적인 최적화가 필요할 수 있습니다.
